## Build Triggers

- **"build database tables and analytics"** → Create all Supabase tables and analytics dashboard
- **"fire it up"** → Build out full working app with real backend

## Feature Priority (when "fire it up")

1. **PRIMARY**: Discipler/Friend/Group devotional accountability — fully functional first
2. **Coming Soon screens**: Family, Podcast, and other newer features get placeholder "Coming Soon" screens
3. Everything should have a screen/presence in the app, but non-primary features just show "Coming Soon"

## Key Context
- Features like different devotional types, podcast, family features were added after initial design
- Those don't exist yet and should NOT be built as functional until explicitly requested
- The core discipler accountability loop is the MVP
